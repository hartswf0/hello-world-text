Feature Layers

surface -> substrate -> effect

HELLO / RigGraph

8 pts
Dead endGenerated geometry and part arrays do not fully explain the rig to another mode.
Base needA copyable <RigGraph> with part ids, transforms, parents, materials, sources, and gameplay tags.
CostHigh because physics, visuals, multiplayer sync, clone, film, and export all touch the rig.

WORLD / WorldGraph

8 pts
Dead endWorlds look different but do not reliably change stealth, scoring, handling, evidence, or objectives.
Base needA <WorldGraph> that includes both geometry and rule payload.
CostHigh because every game mode must read world rules without duplicating world code.

SAY / DIRECT / DirectiveGraph

8 pts
Dead endParser commands can spawn or place things, but cannot author durable bots/enemies/allies as apparatus.
Base needA <DirectiveGraph> that turns language into actors, roles, teams, goals, behaviors, and small structures.
CostHigh because generated actions need safety, validation, preview, failure handling, and undo semantics.

PLAY / GameModeRegistry

13 pts
Dead endEach game mode becomes its own private loop, so missing games cannot be added in clean chunks.
Base needA <GameModeRegistry> that consumes RigGraph, WorldGraph, DirectiveGraph, and EventStream.
CostVery high because mode lifecycle touches UI visibility, physics ticks, scoring, bots, and replay.

FILM / PARIS / EventStream

13 pts
Dead endA film can show pixels but cannot explain collisions, clones, hide checks, orders, or verdicts.
Base needAn <EventStream> that feeds <FilmRecord> and <ParisCase>.
CostVery high because time, identity, camera, mode state, and export must agree.

RAIL / Hidden Apparatus

5 pts
Dead endTools compete with play, and advanced/debug elements become permanent chrome.
Base needA <RailApparatus> with sections: BUILD, DIRECT, FILM, CASE, SYSTEM.
CostMedium because it is mostly information architecture, but every surface needs a clean home.

Deadends And Culdesacs

what blocks the base
Geometry-only HELLOLooks successful

Builds meshes but does not expose stable rig semantics to game modes.

Exit: promote every part into <RigGraph> records before rendering.

Decor-only WORLDMain culprit

World prompts affect color/props but do not alter friction, sight, hiding, evidence, or scoring.

Exit: make family differences mechanical and testable.

Parser-only DIRECTToo narrow

Commands can run, but they are not validated authored systems.

Exit: produce <DirectiveGraph> with preview/apply/reject.

Mode stubsAccumulating debt

Missing games stay missing because each requires custom state and UI.

Exit: one <GameModeRegistry> contract with vertical slices.

Pixel-only filmWeak record

Video alone cannot support replay, evidence, or verdict.

Exit: pair capture with <EventStream> and graph snapshots.

Floating apparatusUI bloat

AI log/API/debug surfaces pollute play mode and make features feel bloated.

Exit: hide apparatus in rail drawers and mode-gate surfaces.

Most Expensive Differences

integration risk
Identity Model13 pts

Meshes, rig parts, bots, clones, events, and film frames all need the same entity ids.

Without this, every export/replay/case feature becomes a manual reconstruction.

World Rules13 pts

Families must differ by mechanics: sight, traction, cover, noise, hazards, evidence anchors.

This is expensive because physics, AI, UI, and scoring must consume the same rule payload.

AI Contracts8 pts

HELLO and DIRECT need generated JSON that validates before touching the live scene.

This is expensive because invalid AI output must fail safely and still teach the user what changed.

Mode Lifecycle13 pts

Sneak, film, Paris, CTF, golf, free drive, and future games need setup/tick/teardown/export.

This is expensive because UI gating, camera, scoring, bots, and event recording all meet here.

Event Time8 pts

Film and Paris need a durable timeline of actions, collisions, zones, clone births, and claims.

This is expensive because current behavior is live-state oriented, not history oriented.

Rail IA5 pts

The advanced apparatus needs one map so features do not sprawl across the screen.

This is cheaper than graphs, but it is necessary before the system feels coherent.
