# OPERATOR 14 — THE GALLOP

Single file: `operator-studio-14.html`. Protocol unchanged from OP-12/13 — peers on
OP-13 still interoperate. Worker deployment note at the end still applies for >2 peers.

## Purpose

Kill the air-traffic-control boot. OP-14 opens as a board, a pen, and a ghost horse.
Every panel is earned by performing its verb once; by the time the full board exists
the operator has already authored, branched, directed, and published. Crew chrome does
not exist in the DOM until a second human does. The interface is the tutorial, and the
tutorial is the founding experiment of cinema: Stanford's 1878 bet, settled by playback.

## Entities

- **Gate bitmask** — `GT_TL=1, GT_PLAY=2, GT_TABS=4, GT_D=8, GT_E=16` (`GT_ALL=31`),
  persisted at `localStorage['operator.gate']`. Body classes `gt-tl … gt-e` + `.onb`
  drive pure-CSS visibility. `applyGates()` is the single writer.
- **ONB** — `{active, pose, ids[4], mask, stage: trace → toplay → bet → null}`.
- **Horse** — `horsePose(i)`: four gallop key poses rasterized from named segments
  (`_hline`, `_leg`) into a 128×96 mask at runtime; zero assets. Ground is a dashed
  fact at y=80. Pose index 2 is the suspension — verified by test: zero ink in the
  contact band (rows 77–79) while every other pose touches it.
- **Lens** — `lensSrc ∈ {world, self, file, crew}`. One shutter (`capGrab`,`capPush`),
  four kinds of light. `crew` reads `themGrid` (the decoded remote feed).
- **Sequence run** — consecutive timeline cells sharing `kind ∈ SEQ_KINDS` and a
  `batch` id stamped by `appendSequence`. Runs render as strips (sprocket band +
  provenance color rail + `CLIP ×N` flag); lone frames stay cel-cards.
- **Node.rev** — repaint currency. Bumped by `editNode` and inbound `F` frames;
  read by the recycled strip and the BRANCH atlas.

## Operations — the tour is the tutorial

The gallop is now a full production tour, not just an unlock ladder. The hud directs
every step in order, and a **call sheet** under it tracks seven marks:
`A ▶ ? B D E C` — done marks fill white, the current mark burns amber.

| Mark | Gate/stage | Trigger — performed, not read | Materializes / directs |
|---|---|---|---|
| A | GT_TL | ≥110 ink dots inside the ghost, ≤2× outside, on pointer-up; poses 2–4 auto-advance (SEE → PREDICT → CHOOSE) | timeline strip |
| ▶ | GT_PLAY | 4 accepted poses | PLAY, pulsing |
| ? | GT_TABS | press PLAY, then tap the suspension frame (`ONB.ids[2]`) | tab bar; B pulses; hud → "MAKE AN ALTERNATE TAKE" |
| B | GT_D | first fork — any parent's kids reach 2 (`mkNode` hook) | D · DIRECT; hud → "THE LIGHT TABLE" |
| D | GT_E | first DIRECT commit (`ltPersist` / `ltCommitQuiet`) | E · IN/OUT; hud → "SHIP IT" |
| E | stage `publish` | any export/publish press in E (capture-phase listener on `panel-save`) | hud → "TWELVE CAMERAS BEAT ONE", C pulses |
| C | stage `crew` | opening the crew flow, or an actual room | tour complete: "YOU SPEAK CINEOSIS" |

**Tour latch** (`operator.tour`) is separate from the gates: gates say what is
*visible*, the tour says whether the hud still directs. Mid-tour reloads **resume**
at fork / direct / publish from the gate state. Post-bet stages carry a dismiss (×).
Bypass: invite link, any live room, the skip link, or a loaded working board — each
completes both latches; a crew's board is never gated. Replay in HELP clears both.

**Joining is oriented, not gated.** The help sheet no longer auto-opens at boot (it
used to pop over the guest join flow — that was the thing in the way). When a room
first goes live, a one-shot hud orients each side: guest — "YOU ARE ON THE CREW —
the board is shared; your ink carries your name"; host — "CREW CHANNEL OPEN — they
land ungated, camera optional." Auto-hides in 9s.

## Unified lens & chrome

Face group deleted; capability preserved: *drop local face* = LENS→SELF→shutter,
*drop remote face* = LENS→CREW→shutter. **Wipe** moved into the board tool row.
Top bar solo = camera mark + help only; `#connect` is gone from the bar (C · CREW is
the one door: it opens the room flow solo, the crew panel when live, and the panel
gained a primary **Open a room / invite crew** button). Mic, chat, and the presence
strip exist only under `body.crewed`. Log demoted to the HELP footer.
The help sheet is retitled **Field reference** — it never opens itself; the gallop
teaches, the reference answers. Its first section is THE PRODUCTION ORDER.

## Performance directives (measured causes → fixes)

1. `renderStrip` rebuilt every cell with a fresh canvas, ~6 listeners, and
   `scrollIntoView` inside the loop → **recycled keyed cells** (repaint only on
   `rev` change), **one delegated listener set**, **one rAF-deferred centering**.
2. `gotoNode` fanned to seven renders including closed panels → **render governor**:
   BRANCH/DIRECT no-op to a dirty flag while closed; the tab handlers already flush
   on open.
3. `renderBranches` allocated a throwaway canvas per SHOT per render → **thumb
   atlas** (128 slots, rev-keyed, clock eviction), zero per-render allocation.
4. `.tlc { contain: layout paint }`; a throttled `PERF` line in LOG reports
   nodes · cells · new · ms, so slowdowns are read, not felt.

## Invariants

Single file, no runtime deps; BEFLIX-128 untouched; all loops carry named ceilings
(`TRACE_MIN`, `ONB_POSES`, `ATLAS_SLOTS`, guard counters); no emoji glyphs — every
mark is vector or CSS; no border-radius on rectangular containers; a guest arriving
by link is never gated; the shutter is source-blind; renames never touch ids, kinds,
or the control protocol.

## Failure modes handled

Ghost vs. sloppy tracing (2× outside tolerance, erase to retry); wrong bet tap
(hint, not failure); localStorage denied (gates default open only via skip/room, game
simply replays); crew feed absent (CREW chip hides, shutter refuses with a reason);
legacy Worker and camera-optional joins carry over from OP-13 unchanged.

## Test ladder

`node --check` both blocks · id cross-check (all refs null-guarded or present) ·
emoji scan clean · radius audit clean (two pre-existing circles) · VM smoke:
horse poses dense + distinct, suspension band empty while 0/1/3 touch ground,
gate bitmask accumulation, run detection incl. batch split and single-frame
exclusion, trace acceptance math, atlas slot stability + rev repaint + eviction.

VM integration drive of the tour machine: trace ×4 → toplay → bet (wrong tap held,
suspension tap advanced) → fork → direct → publish → ship click → crew → open-connect
completes the latch; silent re-init when done; resume at fork/direct/publish from
gates; loaded-board safety (tree > 1 + fresh gates = ungated + tour done).

Manual pass for the phone: cold boot → the full seven marks through publish and crew;
skip link; tour dismiss (×) post-bet; invite-link lands straight in the join flow with
no help modal; crew hello on both sides; LENS world/self/file/crew; crew chrome only
with a room.

## Deployment note (unchanged from OP-13)

`worker.js` (mesh Part A: roster, `to` routing, `MAX_OPERATOR_PEERS=6`) must be
deployed to the Cloudflare Worker for rooms of more than two operators. Two-peer
rooms degrade gracefully on the legacy Worker via the roster latch + pairing kick.
