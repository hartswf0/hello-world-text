CHAPTER 1 · LITERATURE / ACADEMIA

The Worldtext Problem

More Than the Eye

TOP

Generative abundance converts production scarcity into selection scarcity: one visible sample stands in for an uninspectable distribution.

Primary encoder: Literature / Academia Secondary pressure: Formal systems

The chapter is fundamentally a media-philosophical problem about representation, attention, possible worlds, and the politics of selection; probability supplies its formal undertow.

OPEN FULL LINEAGE

Ancestral lineage

  • Jorge Luis Borges — “The Library of Babel”: total textual possibility as existential blindness.
  • G. W. Leibniz — possible worlds: actuality as one realized selection among alternatives.
  • Claude Shannon — information theory: uncertainty and distributions become measurable structures.
  • Herbert Simon — bounded rationality: abundance produces a scarcity of attention.
  • Nelson Goodman — Ways of Worldmaking: worlds are made through symbolic operations.
  • Umberto Eco — The Open Work: interpretive multiplicity without complete survey.
  • Lev Manovich — database logic: selection replaces fixed sequence as a dominant cultural form.

Origin conditions

Questions

  • When does abundance become opacity?
  • What changes when choosing becomes the main creative labor?

Conflicts

  • scarcity / abundance
  • production / curation
  • freedom / blindness

Scenes

  • the library without an edge
  • the search-results page
  • the reroll button
  • the contact sheet no one can finish

Methods

  • media genealogy
  • conceptual analysis
  • interface criticism
  • sampling experiments
  • attention-budget studies

Treat attention as the scarce material, rerolling as a labor practice, and selection as an act of governance. Use prompt logs, discarded outputs, and stopping decisions as primary evidence. End by proposing a bounded-inspection protocol that makes absence explicit rather than claiming exhaustive visibility.

CHAPTER 2 · LITERATURE / ACADEMIA

The Genealogy

The Object That Flinches

TOP

Text ceases merely to describe an object and becomes an operative address that helps produce, traverse, and alter what it names.

Primary encoder: Literature / Academia Secondary pressure: Software studies

The chapter tracks a conceptual migration across textuality, hypertext, performativity, cybertext, and software: a genealogy is more revealing than a purely technical account.

OPEN FULL LINEAGE

Ancestral lineage

  • J. L. Austin — performative utterances: words can enact rather than merely report.
  • Roland Barthes — readerly and writerly texts: reading redistributes authorship.
  • Vannevar Bush — Memex: reading becomes associative traversal.
  • Ted Nelson — hypertext: textual sequence becomes a navigable structure.
  • Espen Aarseth — Cybertext: reading includes nontrivial operational labor.
  • N. Katherine Hayles — materiality of inscription: textual meaning depends on its technical embodiment.
  • Vilém Flusser and Alexander Galloway — technical images and protocol: symbolic surfaces become programmed operations.

  • distance / participation

  • the hypertext link
  • the command line
  • the prompt box

Methods

RESEARCH / FORMAL / R&D ECHO

Construct a genealogy of operative reading from Austin’s speech acts and Barthes’s writerly text through Bush, Nelson, Aarseth, Hayles, and software studies. Use one multimodal generation sequence as the contemporary archive: show how a phrase first appears to describe an image, then alters its production, then changes the operator’s subsequent language. Distinguish reference, traversal, execution, and cross-conditioning at every stage. Ask what critical distance survives when interpretation is also an input channel. The intervention should define worldtext as the object that is read by being operated and operated by being read.

CHAPTER 3 · CODE & MATH

The Method

Thick Prompting

TOP

A prompt is an incomplete specification completed by learned priors; controlled perturbation and captured run state reveal what remains invariant or drifts.

Primary encoder: Code & Math Secondary pressure: Experimental method

The chapter’s core claims concern representation, reproducibility, experimental control, confounding, and perturbation. Its strongest lineage is formal and methodological.

OPEN FULL LINEAGE

Ancestral lineage

  • R. A. Fisher — experimental design: isolate effects through controlled variation.
  • George Box — models and iterative experimentation: useful approximations must be tested, not worshiped.
  • Bayesian inference — priors complete underdetermined evidence.
  • Software configuration management — reproducibility depends on versions, state, and dependency capture.
  • D’Amour et al. — underspecification in machine learning: many models satisfy the same observed objective while behaving differently.
  • Prompt programming and in-context learning — natural language becomes a soft control surface.
  • Reproducibility and model-card traditions — claims require documented conditions and limitations.

Origin conditions

Questions

  • Which parts of an output come from the instruction and which from priors?
  • What survives a one-variable change?
  • What must be captured for a claim to be rerun?

Representations

  • run tuple (model, prompt, seed, parameters, output)
  • perturbation graph
  • difference matrix
  • failure taxonomy

Invariants

  • one independent variable changes per comparison
  • run metadata remains fixed
  • evaluation target is declared before inspection

Bottlenecks

  • model nondeterminism
  • version drift
  • confounded prompt edits
  • subjective similarity judgments

Failed

  • prompt folklore without logs
  • single-run interpretation
  • seed worship that ignores model and version changes

Epochs of formation

PeriodFormationIntellectual shiftWeight
1920s–1970sDesigned experiments
Fisher, Box		Causal claims depend on controlled perturbation and explicit error.86
1970s–2000sReproducible computation
version control, literate programming, configuration capture		The experimental object expands to include code, dependencies, and environment.82
2010sML credibility and underspecification
benchmarking, model cards, D’Amour et al.		Equivalent scores conceal divergent internal and downstream behavior.95
2020sPrompt-based control
foundation-model interfaces		Natural-language specifications become experiments whose missing clauses are filled by priors.100

Migration map

  • natural-language instruction → experimental variable: prompting becomes testable rather than mystical.
  • seed-only reproducibility → full run capture: the reproducible unit becomes the entire configuration.
  • output critique → perturbation catalog: interpretation becomes a structured study of compliance, drift, substitution, and failure.

Evidence stack

Run records, parameter manifests, controlled prompt families, blinded comparisons, inter-rater agreement, and longitudinal reruns across model versions.

Lineage axis

Primitive objects

prompt clauses; seeds; parameter vectors; outputs; annotations

Transformation rules

perturb one clause; hold configuration; rerun; compare; classify

Success pressures

reproducibility; interpretability; low confounding; useful failure categories

Proof / evaluation logic

replication across seeds, raters, and model versions; preregistered comparisons

Risks

API drift; hidden safety layers; cherry-picked outputs; semantic confounds; evaluator fatigue

Mitigations

content hashes; environment manifests; pairwise blinded review; perturbation budgets; automatic diff summaries

Signature ecology

Problem logic

The model completes missing intention from priors acquired elsewhere.

Representation logic

The irreducible unit is the run tuple, not the prose prompt.

Invariant logic

Only the designated variable may change within a comparison.

Method logic

Perturb, rerun, compare, classify, and preserve the complete lineage.

Evaluation logic

Measure compliance, drift, substitution, brittleness, and reproducibility.

Intervention logic

Turn prompting into an auditable experimental practice.

RESEARCH / FORMAL / R&D ECHO

Design a thick-prompting laboratory around the run tuple (model, conditioning, seed, parameters, output). Represent a prompt as clauses in a perturbation graph, then change exactly one clause per experimental family while holding the rest of the configuration fixed. Borrow Fisher’s discipline of controlled comparison, Box’s iterative skepticism, and the ML literature on underspecification. Build a reproducible recorder, a blinded pairwise evaluator, and a taxonomy for invariant, brittle, substituted, and failed properties. Test the system across seeds and model versions, explicitly treating version drift and hidden platform mediation as threats to validity.

CHAPTER 4 · CODE & MATH

Composition

The Writable Image

TOP

Text-to-image composition is a joint-constraint problem in which locally plausible objects often survive while relations, causality, cardinality, and identity fail.

Primary encoder: Code & Math Secondary pressure: Visual culture

The binding gap is best understood through representations, constraints, attention, scene graphs, and generate-check-repair loops, while visual theory explains why local plausibility remains persuasive.

OPEN FULL LINEAGE

Ancestral lineage

  • Gotthold Ephraim Lessing — Laocoön: media differ in how they organize spatial and temporal relations.
  • W. J. T. Mitchell — word–image relations exceed simple illustration.
  • Anne Treisman — feature integration: binding attributes into coherent objects is a distinct problem.
  • Scene graphs and visual relationship detection — images represented as objects, attributes, and relations.
  • CLIP and contrastive vision-language models — language and images share a learned alignment space.
  • Diffusion models and classifier-free guidance — images emerge through iterative stochastic denoising under conditioning.
  • ControlNet, masks, and spatial conditioning — auxiliary structures attempt to stabilize composition.

Origin conditions

Questions

  • Why do correct parts fail to compose into a correct whole?
  • Which relations are most brittle?
  • How can a writable image expose its unmet constraints?

Representations

  • typed scene graph
  • constraint set
  • mask stack
  • identity ledger
  • render–parse comparison

Invariants

  • object count
  • attribute ownership
  • spatial relation
  • identity persistence
  • causal direction

Bottlenecks

  • attention binding
  • relational complexity
  • text encoder compression
  • weak spatial grounding

Failed

  • longer prompts as universal cure
  • negative prompts without explicit structure
  • judging only surface plausibility

Epochs of formation

  • bag of attributes → scene graph: compositional meaning is relocated into explicit relations.
  • single-pass generation → generate-parse-repair: authorship becomes iterative constraint management.

Evidence stack

Prompt-scene graphs, generated image sets, relation-level annotations, parse mismatches, repair histories, and complexity-stratified pass rates.

Lineage axis

Primitive objects

entities; attributes; predicates; masks; latent states

Transformation rules

encode → denoise → parse → compare → repair

Success pressures

Risks

Signature ecology

Problem logic

The model satisfies visible nouns and adjectives while dropping the relations that made the described world itself.

Representation logic

Encode the requested image as a typed graph, not a prose string alone.

Invariant logic

Every required entity, attribute, relation, count, and identity must remain inspectable.

Method logic

Generate, parse, compare to constraints, localize failure, and repair.

Evaluation logic

Separate local plausibility from global consistency.

Intervention logic

Make the binding gap editable rather than mystifying.

RESEARCH / FORMAL / R&D ECHO

Build a writable-image compositor that compiles prose into a typed scene graph of entities, attributes, cardinalities, spatial predicates, and causal relations. Condition a diffusion model with text, masks, and spatial controls, then parse each result back into the same graph and compute relation-level failures. Draw on Treisman’s binding problem, scene-graph vision, and contemporary controllable generation, but refuse surface plausibility as the sole metric. Add a repair loop that targets only failed predicates while protecting already satisfied invariants. Evaluate performance as relational complexity rises and publish separate pass rates for objects, attributes, counts, identity, space, and causality.

CHAPTER 5 · CODE & MATH

Assembly Is Sampling

The River That Refills

TOP

Generative editing replaces a fixed corpus with a resamplable distribution, turning montage into stochastic search through an expanding derivation graph.

Primary encoder: Code & Math Secondary pressure: Film and media theory

The chapter joins montage theory to sampling algorithms, search, state machines, and provenance. The decisive problem is how to steer an expanding formal space without losing legibility.

OPEN FULL LINEAGE

Ancestral lineage

  • Lev Kuleshov and Sergei Eisenstein — montage: meaning emerges through selection and juxtaposition.
  • Dziga Vertov — the archive and apparatus as active constructors of cinematic reality.
  • Lev Manovich — database cinema: sequence is selected from a structured collection.
  • Monte Carlo methods — approximate exploration through repeated sampling.
  • Markov-chain and sequential decision models — current state shapes the next draw.
  • Evolutionary search and quality-diversity algorithms — candidate populations are generated, selected, and mutated.
  • Mixed-initiative creativity research — humans and systems alternate proposal, evaluation, and revision.

Origin conditions

Questions

  • What is editing when material is generated during the edit?
  • How does a maker know where a sequence came from?
  • When should sampling stop?

Representations

  • derivation DAG
  • candidate pool
  • state machine
  • selection ledger
  • attention budget

Invariants

  • every accepted shot has a parent lineage
  • selection criteria are recorded
  • sampling budget is bounded
  • sequence order is recoverable

Bottlenecks

  • combinatorial explosion
  • path dependence
  • opaque memory
  • curator fatigue

Failed

  • infinite rerolling
  • flat galleries without lineage
  • final sequences detached from rejected alternatives

Epochs of formation

PeriodFormationIntellectual shiftWeight
1910s–1930sMontage as constructive selection
Kuleshov, Eisenstein, Vertov		Editing is understood as the production of meaning through ordered cuts.82
1960s–2000sDatabase and nonlinear editing

Migration map

  • fixed footage → addressable database: editing gains random access.
  • database → distribution: unseen material can be produced on demand.
  • timeline → derivation graph: the work includes the branches, rejections, and state transitions that made the cut.

Evidence stack

Variant families, edit histories, branch graphs, time-to-selection logs, rejected-shot clusters, and comparisons between fixed-corpus and resamplable editing.

Lineage axis

Primitive objects

shots; variants; states; branches; scores; selections

Transformation rules

draw → inspect → select/reject → mutate conditioning → redraw

Success pressures

Evaluation logic

Risks

Signature ecology

Problem logic

The archive expands faster than the editor can understand it.

Representation logic

Assembly is a stochastic state machine recorded as a derivation graph.

Invariant logic

Every visible cut retains its generative and editorial ancestry.

Method logic

Sample under budget, summarize branches, expose state, and preserve reversibility.

Evaluation logic

Judge both the sequence and the legibility of how it came to be.

Intervention logic

Build an editor that makes steering visible while the river refills.

RESEARCH / FORMAL / R&D ECHO

Design a generative editing engine in which each shot is a node in a derivation DAG rather than an isolated file. Combine montage theory with Monte Carlo search and mixed-initiative creativity: the system proposes bounded variant sets, the maker selects or rejects, and each decision updates the next sampling distribution. Record criteria, state changes, rejected branches, and stopping conditions. Visualize both the linear cut and the expanding search history, allowing any accepted choice to be reversed without erasing its descendants. Evaluate narrative coherence together with search cost, branch diversity, curator fatigue, and reconstructability.

The World That Remembers

TOP

Primary encoder: Code & Math Secondary pressure: Narrative systems

The chapter is centered on state, persistence, identity, event history, and causal consistency. These are formal architecture problems with narrative consequences.

Ancestral lineage

  • Database transactions and event sourcing — current state is reconstructed from durable events.
  • Version control — identity persists through diffs, branches, and merges.
  • Object-oriented identity and the Memento pattern — objects preserve recoverable state.
  • Memory-augmented neural networks and retrieval systems — external memory supplements bounded model context.
  • World-model research — agents predict transitions inside compressed environment representations.

Architect a persistent generative world around an event-sourced state ledger. Give every entity a durable identity, every change a typed event, and every consequence an explicit rule for persistence or reversal. Before each generation, compile only the relevant state and constraints into context; afterward, parse proposed changes and reconcile them against causal order, identity, and prior commitments. Draw on statecharts, database transactions, version control, and memory-augmented models. Evaluate the system with contradiction suites, delayed-consequence tests, identity stress tests, and counterfactual branches, while keeping conflict and uncertainty visible to the operator.

Legible Mechanisms

Provenance

TOP

A persuasive generative artifact needs deployable lineage infrastructure that records enough of its partial history to support doubt, audit, attribution, and repair.

Primary encoder: Research & Development Secondary pressure: Archival theory and data infrastructure

Provenance becomes meaningful only when translated from archival principle into interoperable logs, manifests, user interfaces, governance thresholds, and operational adoption.

CHAPTER 8 · CODE & MATH

Legal Worlds

Constraint and Verification

TOP

A world becomes legally buildable only when perceptual output is translated into symbolic structure and tested against explicit constraints by a verifier with known coverage.

Primary encoder: Code & Math Secondary pressure: Design and formal methods

The distinction between looking valid and being valid belongs directly to type systems, grammars, solvers, model checking, simulation, and proof obligations.

OPEN FULL LINEAGE

Ancestral lineage

  • Formal grammars — valid structures generated from explicit production rules.
  • C. A. R. Hoare and Edsger Dijkstra — program correctness expressed through assertions and disciplined construction.
  • Curry–Howard correspondence and type systems — propositions become types and valid constructions become proofs.
  • SAT/SMT solving — complex constraint sets become mechanically checkable.
  • Model checking — finite systems are exhaustively tested against temporal properties.
  • CAD, physics simulation, and digital twins — geometry must survive executable material constraints.
  • Grammar-constrained and neuro-symbolic generation — learned proposal is coupled to symbolic validity.

  • proof report

Bottlenecks

  • perception-to-symbol translation
  • solver complexity
  • incomplete physics
  • specification errors

Failed

  • opaque pass/fail badges

Borrow from Hoare logic, model checking, CAD, and neuro-symbolic systems, but require the verifier to publish its coverage and blind spots. Return counterexamples that localize failed relations and drive targeted repair rather than indiscriminate regeneration. Evaluate perceptual plausibility, parse fidelity, symbolic validity, and executable success as separate layers, including adversarial cases designed to fool the checker.
